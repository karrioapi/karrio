Carrier connections allow you to perform live rates fetching, label generation,
shipment tracking and any other carrier supported operations. The API allow you to
retrieve the list of carrier accounts configured and accessible from your account.

        <td>/v1/carriers</td>
      </tr>

          /v1/carriers/<strong>[carrier_name]</strong>/services
        </td>
      </tr>
    </tbody>
  </table>

:::info
This API is read-only and does not allow you to create, update or delete carrier connections.
To programmatically create, update or delete carrier connections, use the [GraphQL management API](/reference/management/connections).
:::


## Get carrier connections

<div id="tabs">
<Tabs>
<TabItem value="curl" label="Curl">

```bash
curl --request GET \
  --url https://api.karrio.io/v1/carriers \
  --header 'Authorization: Token [API_KEY]'
```

</TabItem>
<TabItem value="javascript" label="Javascript">

```javascript
(async () => {
  const carriers = await karrio.carriers.list();
})();
```

</TabItem>
</Tabs>
</div>

Karrio returns a list for carrier settings object in response to your API request.

## Get carrier services


<div id="tabs">
<Tabs>
<TabItem value="curl" label="Curl">

```bash
curl --request GET \
  --url https://api.karrio.io/v1/carriers/canadapost/services \
  --header 'Authorization: Token [API_KEY]'
```

</TabItem>
<TabItem value="javascript" label="Javascript">

```javascript
(async () => {
  const services = await karrio.carriers.get_services("canadapost");
})();
```

</TabItem>
</Tabs>
</div>

Karrio returns a dictionary of carrier service_code as key and
service_name as values.
